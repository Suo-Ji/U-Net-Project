## 1. 项目简介

本项目基于 **U-Net** 架构，引入 **EdgeSE 边缘注意力机制**，在编码器阶段利用不同边缘检测算法（Canny / Sobel / Laplacian）生成的边缘图引导特征学习，从而提升道路分割精度。在 **CamVid** 和 **Cityscapes** 两个数据集上进行道路二值分割实验，四模型统一训练、公平对比。

**四个实验模型：**

| 模型名称 | 架构 | 边缘检测 | 说明 |
|---|---|---|---|
| `unet` | 标准 U-Net | 无 | 基线模型，不使用边缘信息 |
| `unet_canny` | U-Net + EdgeSE | Canny | Canny 边缘检测引导注意力 |
| `unet_sobel` | U-Net + EdgeSE | Sobel | Sobel 梯度边缘引导注意力 |
| `unet_laplacian` | U-Net + EdgeSE | Laplacian | Laplacian 二阶边缘引导注意力 |

**核心功能：**

| 功能 | 说明 |
|---|---|
| 四模型对比 | 标准U-Net + 三种边缘注意力变体，使用统一训练框架，公平对比 |
| 双数据集支持 | CamVid 和 Cityscapes 独立训练与评估 |
| 11 项评估指标 | Loss、IoU、Pixel Acc、Precision、Recall、F1、NLL、ECE、显存、推理速度、训练耗时 |
| 四模型同时对比可视化 | 训练曲线叠加、预测结果并排、终端指标对比表，四模型一目了然 |
| 自定义图片推理 | 对任意图片进行道路分割预测，支持批量预测 |

**技术栈：** PyTorch + Albumentations + OpenCV + Matplotlib

## 4. 快速上手

### 训练模型（8 种组合）

```bash
# CamVid 数据集
python main.py camvid unet                # CamVid + 标准 U-Net（基线）
python main.py camvid unet_canny          # CamVid + U-Net + Canny 边缘注意力
python main.py camvid unet_sobel          # CamVid + U-Net + Sobel 边缘注意力
python main.py camvid unet_laplacian      # CamVid + U-Net + Laplacian 边缘注意力

# Cityscapes 数据集
python main.py cityscapes unet            # Cityscapes + 标准 U-Net
python main.py cityscapes unet_canny      # Cityscapes + U-Net + Canny
python main.py cityscapes unet_sobel      # Cityscapes + U-Net + Sobel
python main.py cityscapes unet_laplacian  # Cityscapes + U-Net + Laplacian
```

执行后自动完成：数据加载 → 边缘图计算 → 训练 30 个 Epoch → 保存最佳模型 → 绘制训练曲线 → 验证集预测可视化 → 保存训练历史 JSON。

### 测试已有模型


### 四模型的数据流对比

**标准 U-Net（基线）：**

```
输入图像 (B, 3, 256, 256)
        │
        ▼
┌──────────────────────────────────────┐
│  编码器：64 → 128 → 256 → 512 → 1024 │
│     ↓      ↓      ↓      ↓           │
│     └──特征图拼接(Concat)─┘           │
│  解码器：1024 → 512 → 256 → 128 → 64 │
│        (ConvTranspose2d 上采样)       │
└──────────────────────────────────────┘
        │
        ▼ 输出 Logits (B, 1, 256, 256)
```

**U-Net + EdgeSE（边缘注意力变体）：**

```
输入图像 (B, 3, 256, 256)     边缘图 (B, 1, 256, 256)
        │                            │
        ▼                            │ (Canny / Sobel / Laplacian)
┌───────────────────────────┐        │
│  inc: 3 → 64              │        │
│        │                  │        │
│        ▼                  │        │
│  ┌──────────────────┐     │        │
│  │  EdgeSE 模块      │◄────┘        │
│  │  GAP → FC → σ    │              │
│  │  通道注意力加权    │              │
│  └──────────────────┘              │
│        │                           │
│  编码器: 64 → 128 → 256 → 512 → 1024│
│     ↓      ↓      ↓      ↓         │
│     └──特征图拼接(Concat)─┘         │
│  解码器: 1024 → 512 → 256 → 128 → 64│
│        (ConvTranspose2d 上采样)     │
└───────────────────────────────────┘
        │
        ▼ 输出 Logits (B, 1, 256, 256)
```

### 6.2 model.py — 模型定义（U-Net + 边缘注意力变体）

包含三个核心组件和一个工厂函数。

#### 基础组件

| 组件 | 功能 |
|---|---|
| `DoubleConv` | Conv2d(3×3) → BN → ReLU → Conv2d(3×3) → BN → ReLU |
| `Down`（编码器） | MaxPool(2×2) → DoubleConv |
| `Up`（解码器） | ConvTranspose2d(2×2) → 拼接跳跃连接 → DoubleConv |

#### UNet — 标准 U-Net

| 阶段 | 层 | 输入通道 | 输出通道 | 空间尺寸 |
|---|---|---|---|---|
| 编码器 | `inc` | 3 | 64 | 256×256 |
| 编码器 | `down1` | 64 | 128 | 128×128 |
| 编码器 | `down2` | 128 | 256 | 64×64 |
| 编码器 | `down3` | 256 | 512 | 32×32 |
| 瓶颈层 | `down4` | 512 | 1024 | 16×16 |
| 解码器 | `up1` | 1024 | 512 | 32×32 |
| 解码器 | `up2` | 512 | 256 | 64×64 |
| 解码器 | `up3` | 256 | 128 | 128×128 |
| 解码器 | `up4` | 128 | 64 | 256×256 |
| 输出 | `outc` | 64 | 1 | 256×256 |

**forward 输入：** `x` — 图像张量 `(B, 3, H, W)`

#### EdgeSE — 边缘注意力模块

采用 SE-Block（Squeeze-and-Excitation）风格，用边缘图引导编码器特征的通道权重：

```
edge_map (B, 1, H, W)
    → 全局平均池化 (GAP) → (B, 1)
    → FC(1 → C//4) → ReLU → FC(C//4 → C) → Sigmoid
    → 通道权重 (B, C, 1, 1)

feature_map (B, C, H, W) × 通道权重 → 加权特征图
```

其中 `C = 64`（第一层编码器通道数），`reduction = 4`（压缩比）。

#### UNetEdgeAttention — U-Net 边缘注意力变体

继承自 `UNet`，在编码器 `inc` 层之后插入 `EdgeSE` 模块：

```python
x1 = self.inc(x)              # (B, 64, H, W) 编码器第一层
x1 = self.edge_se(x1, edge_map)  # 边缘注意力加权
# 后续编码器和解码器与标准 U-Net 相同
```

**forward 输入：** `x` — 图像张量 `(B, 3, H, W)`，`edge_map` — 边缘图 `(B, 1, H, W)`

#### 四模型核心区别

| 对比项 | U-Net | U-Net+Canny | U-Net+Sobel | U-Net+Laplacian |
|---|---|---|---|---|
| 边缘检测 | 无 | Canny (双阈值) | Sobel (梯度幅值) | Laplacian (二阶导数) |
| EdgeSE 模块 | 无 | ✅ | ✅ | ✅ |
| 额外输入 | 仅图像 | 图像 + Canny 边缘图 | 图像 + Sobel 边缘图 | 图像 + Laplacian 边缘图 |
| 参数量 | 31,037,633 | ≈31,040,001（多 EdgeSE 参数） | ≈31,040,001 | ≈31,040,001 |

**三种边缘检测算法的特点：**

| 算法 | 原理 | 输出特征 |
|---|---|---|
| **Canny** | 高斯平滑 → 梯度计算 → 非极大值抑制 → 双阈值检测 | 细线边缘，噪声少，二值化输出 |
| **Sobel** | 水平和垂直方向 3×3 差分卷积 → 梯度幅值 | 较粗边缘，保留梯度强度信息 |
| **Laplacian** | 二阶导数卷积核检测零交叉点 | 细节丰富但对噪声更敏感 |

#### 工厂函数 `get_model_class(model_name)`

```python
from model import get_model_class

ModelClass = get_model_class("unet_canny")      # 返回 UNetEdgeAttention
ModelClass = get_model_class("unet_sobel")      # 返回 UNetEdgeAttention

```

### 6.3 dataset.py — 数据集加载器

包含两个数据集类、边缘图计算函数和一个工厂函数。

#### CamVidDataset

- 目录映射：图像 `{split}/` → 标签 `{split}annot/`，文件同名
- 标签处理：灰度索引掩膜，Road=**3** → 1.0

#### CityscapesDataset

- 目录映射：`leftImg8bit/{split}/{city}/` → `gtFine/{split}/{city}/`
- 文件名映射：`*_leftImg8bit.png` → `*_gtFine_labelIds.png`
- 标签处理：灰度索引掩膜，Road=**7** → 1.0

#### 边缘图计算 `compute_edge_map(image_rgb, edge_type)`

从 RGB 图像实时计算边缘图，用于 EdgeSE 模块：

| edge_type | 算法 | 实现细节 |
|---|---|---|
| `"canny"` | OpenCV Canny | `cv2.Canny(gray, 50, 150)`，双阈值检测 |
| `"sobel"` | OpenCV Sobel | `sqrt(Sobel_x² + Sobel_y²)`，梯度幅值 |
| `"laplacian"` | OpenCV Laplacian | `abs(cv2.Laplacian(gray))`，二阶导数绝对值 |
| `None` | 全零 | 标准 U-Net 不使用边缘图 |

所有边缘图均归一化到 `[0, 1]` 范围，shape 为 `(H, W)` float32。

**边缘类型自动推断：** 根据模型名称自动确定（如 `unet_canny` → `edge_type="canny"`，`unet` → `edge_type=None`）。

#### 共同特性

- 图像预处理：Resize → 数据增强（仅训练时）→ Normalize → ToTensor
- 归一化：mean=(0.5, 0.5, 0.5), std=(0.5, 0.5, 0.5)
- 输出格式：图像 `(3, 256, 256)` float32，掩膜 `(1, 256, 256)` float32，边缘图 `(1, 256, 256)` float32

#### 数据增强管道（仅训练阶段）

| 增强方法 | 参数 | 触发概率 |
|---|---|---|
| `Resize` | 256×256 | 1.0 |
| `HorizontalFlip` | — | 0.5 |
| `RandomRotate90` | — | 0.5 |
| `RandomCrop` | 256×256 | 0.3 |
| `Affine` | 平移±10%, 缩放0.85-1.15, 旋转±15° | 0.5 |
| `ColorJitter` | 亮度/对比度±0.2, 饱和度±0.2, 色调±0.1 | 0.5 |
| `GaussianBlur` | 核 3-5 | 0.2 |
| `GaussNoise` | — | 0.2 |
| `Normalize` | mean=(0.5,0.5,0.5), std=(0.5,0.5,0.5) | 1.0 |
| `ToTensorV2` | — | 1.0 |

### 6.4 losses.py — 损失函数与评估指标

#### 损失函数

| 损失 | 公式 | 说明 |
|---|---|---|
| `DiceLoss` | 1 - 2×\|P∩G\| / (\|P\|+\|G\|) | 对类别不平衡更鲁棒 |
| `CombinedLoss` | 0.5×BCE + 0.5×Dice | 默认使用的组合损失 |

#### 评估指标（共 7 个函数 + 1 个聚合函数）

| 函数 | 指标 | 公式 | 说明 |
|---|---|---|---|
| `compute_iou` | IoU | \|P∩G\| / \|P∪G\| | 交并比 |
| `compute_pixel_accuracy` | Pixel Acc | 正确像素数 / 总像素数 | 像素准确率 |
| `compute_precision` | Precision | TP / (TP+FP) | 预测为道路中真正是道路的比例 |
| `compute_recall` | Recall | TP / (TP+FN) | 真实道路被正确找出的比例 |
| `compute_f1` | F1-Score | 2×P×R / (P+R) | Precision 和 Recall 的调和平均 |
| `compute_nll` | NLL | -mean[y·log(p)+(1-y)·log(1-p)] | 负对数似然，衡量概率预测质量 |
| `compute_ece` | ECE | Σ(n_b/N)·\|acc_b - conf_b\| | 期望校准误差，衡量置信度校准程度 |
| `compute_all_metrics` | — | — | 一次性计算以上全部指标，避免重复二值化 |

所有指标函数的输入均为 **logits**（未经 sigmoid），函数内部自动执行 sigmoid + 二值化。

#### 对比模式执行流程

```
1. 遍历 config.VALID_MODELS 中全部 4 个模型
2. 检查每个模型的权重文件是否存在
3. 加载所有模型到设备

8. 绘制预测结果并排对比图（6 列 × 3 行）
```

## 7. 数据集说明

### CamVid 数据集

CamVid（Cambridge-driving Labeled Video Database）是道路场景语义分割的标准基准数据集。

**目录结构：**

```
CamVid/
├── train/          # 367 张训练图像 (RGB, 360×480)
├── trainannot/     # 367 张训练标签 (灰度索引图)
├── val/            # 101 张验证图像
├── valannot/       # 101 张验证标签
├── test/           # 233 张测试图像
└── testannot/      # 233 张测试标签
```

**标签格式：** 单通道灰度索引图，Road = **3**。

**下载地址：**
- Kaggle: https://www.kaggle.com/datasets/carlolepelaars/camvid
- 原始来源: http://mi.eng.cam.ac.uk/research/projects/VideoRec/CamVid/

### Cityscapes 数据集

Cityscapes 是大规模城市场景语义理解数据集，包含 50 个城市的街景图像。

**目录结构：**

```
Cityscapes/
├── leftImg8bit/
│   ├── train/   (2975 张，27 个城市子目录)
│   ├── val/     (500 张，3 个城市子目录)
│   └── test/    (1525 张，6 个城市子目录)
└── gtFine/
    ├── train/
    ├── val/
    └── test/
```

**标签格式：** `_gtFine_labelIds.png`，Road = **7**。

**注意：** Cityscapes 测试集 GT 不公开完整标注，预测可视化使用验证集。

**下载地址：** https://www.cityscapes-dataset.com/

## 8. 评估指标说明

### 像素级分割质量指标

| 指标 | 公式 | 越大越好 | 说明 |
|---|---|---|---|
| **IoU** | \|P∩G\| / \|P∪G\| | ✅ | 预测与真实重叠区域占并集的比例 |
| **Pixel Accuracy** | 正确像素数 / 总像素数 | ✅ | 整体分类正确率 |
| **Precision** | TP / (TP+FP) | ✅ | 预测为道路的像素中，真正是道路的比例 |
| **Recall** | TP / (TP+FN) | ✅ | 真实道路像素中，被正确找出的比例 |
| **F1-Score** | 2×P×R / (P+R) | ✅ | Precision 和 Recall 的调和平均 |

### 概率校准与损失指标

| 指标 | 公式 | 越小越好 | 说明 |
|---|---|---|---|
| **Loss** | 0.5×BCE + 0.5×Dice | ✅ | 组合损失，衡量训练收敛质量 |
| **NLL** | -mean[y·log(p)+(1-y)·log(1-p)] | ✅ | 负对数似然，衡量输出概率与真实标签的匹配程度 |
| **ECE** | Σ(n_b/N)·\|acc_b - conf_b\| | ✅ | 期望校准误差，衡量模型置信度是否与实际准确率一致。将预测按置信度分桶，比较每桶的平均置信度与实际准确率 |

### 模型效率指标

| 指标 | 单位 | 说明 |
|---|---|---|
| **GPU 显存占用** | MB | 推理时 GPU 峰值显存（`torch.cuda.max_memory_allocated()`） |
| **推理速度** | FPS | 每秒处理图片数（50 个 batch 取平均） |
| **单 epoch 训练耗时** | 秒 | 每轮训练+验证的总耗时 |
