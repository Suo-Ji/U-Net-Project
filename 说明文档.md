## 1. 项目简介

本项目实现了一个基于 **U-Net** 的像素级道路语义分割系统。使用 **CamVid 数据集** 进行训练，将图像中的"道路（Road）"区域与其他区域进行**二值分割**。

**核心功能：**

| 功能 | 说明 |
|---|---|
| 模型训练 | 完整的训练流程，含验证评估、学习率衰减、最佳模型保存 |
| 训练曲线可视化 | 自动生成 Loss / IoU / Pixel Accuracy 曲线图 |
| 测试集评估 | 随机抽取测试图像，并排展示 Input / GT / Prediction |
| 自定义图片推理 | 对任意图片进行道路分割预测，支持拖入即预测 |

**技术栈：** PyTorch + Albumentations + Matplotlib

### 6.2 model.py — U-Net 模型

实现标准 U-Net 架构（Ronneberger et al., 2015），由三个基础组件构成：

#### 组件一：`DoubleConv` — 双卷积块

```
输入 → Conv2d(3×3) → BatchNorm → ReLU → Conv2d(3×3) → BatchNorm → ReLU → 输出
```

每个卷积块包含两次 3×3 卷积 + 批归一化 + ReLU 激活，是 U-Net 的基本构建单元。

#### 组件二：`Down` — 编码器下采样块

```
输入 → MaxPool(2×2) → DoubleConv → 输出
```

通过最大池化将空间尺寸减半，同时通道数翻倍，提取更高层次的语义特征。

#### 组件三：`Up` — 解码器上采样块

```
           ┌──── 跳跃连接（编码器特征）────┐
           │                               │
输入 → ConvTranspose2d(2×2, stride=2) → 拼接(concat) → DoubleConv → 输出
```

通过转置卷积将空间尺寸加倍，与编码器对应层的特征图在通道维度上拼接（跳跃连接），再经双卷积融合。包含尺寸自适应 padding，处理输入尺寸非 2 的幂次的情况。

#### 完整网络 `UNet`

| 阶段 | 层 | 输入通道 | 输出通道 | 空间尺寸 |
|---|---|---|---|---|
| 编码器 | `inc` | 3 | 64 | 256×256 |
| 编码器 | `down1` | 64 | 128 | 128×128 |
| 编码器 | `down2` | 128 | 256 | 64×64 |
| 编码器 | `down3` | 256 | 512 | 32×32 |
| 瓶颈层 | `down4` | 512 | 1024 | 16×16 |
| 解码器 | `up1` | 1024 | 512 | 32×32 |
| 解码器 | `up2` | 512 | 256 | 64×64 |
| 解码器 | `up3` | 256 | 128 | 128×128 |
| 解码器 | `up4` | 128 | 64 | 256×256 |
| 输出 | `outc` | 64 | 1 | 256×256 |

**总参数量：** 约 31,037,633（31M）

**构造参数：**

| 参数 | 默认值 | 说明 |
|---|---|---|
| `in_channels` | 3 | 输入图像通道数（RGB） |
| `out_channels` | 1 | 输出通道数（二值分割） |
| `num_filters` | 64 | 首层滤波器数量 |

### 6.3 dataset.py — 数据集加载器

#### 类：`CamVidDataset`

继承 `torch.utils.data.Dataset`，负责加载 CamVid 数据、应用增强、并将多类别标签转换为二值道路掩膜。

#### 构造参数

| 参数 | 说明 |
|---|---|
| `data_dir` | 数据集根目录路径 |
| `split` | 数据划分：`"train"` / `"val"` / `"test"` |
| `img_size` | 输出图像尺寸，默认 `(256, 256)` |
| `augment` | 是否启用数据增强（仅训练时为 `True`） |

#### 目录映射规则

图像目录 `{split}` 对应标签目录 `{split}annot`：

```
split="train" → train/ + trainannot/
split="val"   → val/   + valannot/
split="test"  → test/  + testannot/
```

图像与标签文件名一一对应（同名文件）。

#### 标签处理流程

```
原始标签（灰度索引图，值 0-11）
        │
        ▼  _mask_to_binary()
二值掩膜（道路像素=1.0, 非道路=0.0）
        │
        ▼  unsqueeze(0)
输出张量 shape: (1, H, W)
```

CamVid 类别索引中 **Road = 3**，将其余所有类别归为非道路。

#### 数据增强管道（仅训练阶段）

按顺序应用以下增强策略：

| 增强方法 | 参数 | 触发概率 | 说明 |
|---|---|---|---|
| `Resize` | 256×256 | 1.0 | 统一尺寸 |
| `HorizontalFlip` | — | 0.5 | 水平翻转 |
| `RandomRotate90` | — | 0.5 | 90° 随机旋转 |
| `RandomCrop` | 256×256 | 0.3 | 随机裁剪 |
| `Affine` | 平移±10%, 缩放0.85-1.15, 旋转±15° | 0.5 | 仿射变换 |
| `ColorJitter` | 亮度/对比度±0.2, 饱和度±0.2, 色调±0.1 | 0.5 | 颜色扰动 |
| `GaussianBlur` | 核 3-5 | 0.2 | 高斯模糊 |
| `GaussNoise` | — | 0.2 | 高斯噪声 |
| `Normalize` | mean=(0.5,0.5,0.5), std=(0.5,0.5,0.5) | 1.0 | 归一化至 [-1, 1] |
| `ToTensorV2` | — | 1.0 | 转为 PyTorch Tensor |

验证和测试阶段仅应用 `Resize` → `Normalize` → `ToTensorV2`，不做增强。

#### `__getitem__` 返回值

| 返回值 | 类型 | 形状 | 值域 |
|---|---|---|---|
| `image_tensor` | `torch.float32` | `(3, 256, 256)` | [-1, 1] |
| `mask_tensor` | `torch.float32` | `(1, 256, 256)` | {0.0, 1.0} |

### 6.4 losses.py — 损失函数与指标

#### `DiceLoss` — Dice 损失

衡量预测与真实标签的重叠程度，对类别不平衡更鲁棒。

```
Dice Score = 2 × |Pred ∩ GT| / (|Pred| + |GT|)
Dice Loss  = 1 - Dice Score
```

- 输入：模型输出的 **logits**（未经 sigmoid），函数内部自动执行 sigmoid
- `smooth` 参数（默认 1e-6）防止除零

#### `CombinedLoss` — 组合损失（默认使用）

```
Total Loss = 0.5 × BCE Loss + 0.5 × Dice Loss
```

- **BCEWithLogitsLoss**：像素级二分类交叉熵，提供基础分类监督
- **DiceLoss**：区域级重叠度量，缓解道路/非道路像素数量不均衡

权重比例可通过 `bce_weight` 和 `dice_weight` 参数调整。

#### `compute_iou` — 交并比

```
IoU = |Pred ∩ GT| / |Pred ∪ GT|
```

- 输入：logits + GT 掩膜
- 内部执行 sigmoid + 阈值 0.5 二值化
- 返回批次平均 IoU

#### `compute_pixel_accuracy` — 像素准确率

```
Pixel Accuracy = 正确预测的像素数 / 总像素数
```

- 同样基于 sigmoid + 0.5 阈值
- 返回批次平均准确率

### 6.5 train.py — 训练与验证

#### `create_dataloaders()`

创建三个 DataLoader：

| DataLoader | 数据集 | Batch Size | Shuffle | 增强开关 |
|---|---|---|---|---|
| `train_loader` | train (367) | 8 | ✅ | ✅ |
| `val_loader` | val (101) | 8 | ❌ | ❌ |
| `test_loader` | test (233) | 1 | ❌ | ❌ |

#### `train_one_epoch(model, train_loader, criterion, optimizer, device)`

单轮训练流程：

```
遍历每个 batch:
    1. 图像、掩膜送入 GPU
    2. 前向传播 → 计算 Loss
    3. 反向传播 → 梯度清零 → 更新参数
    4. 计算当前 batch 的 IoU、Pixel Accuracy
    5. 累积 Loss、IoU、Accuracy

返回: 样本平均 Loss, 样本平均 IoU, 样本平均 Accuracy
```

使用 `tqdm` 显示实时训练进度条。

#### `validate(model, val_loader, criterion, device)`

单轮验证流程，与训练类似但不更新参数（`@torch.no_grad()` + `model.eval()`）。

#### `train(model, device)` — 完整训练流程

```
1. 创建 DataLoader
2. 初始化损失函数、Adam 优化器、ReduceLROnPlateau 调度器

3. for epoch = 1 to 30:
       训练一个 epoch → 得到 train_loss
       验证一个 epoch → 得到 val_loss, val_iou, val_acc
       学习率衰减（监控 val_loss，patience=5，factor=0.5）
       若 val_loss 为历史最优 → 保存模型到 best_model.pth
       打印本轮指标

4. 返回: model, test_loader, history
```

**history 字典结构：**

```python
{
    "train_loss": [float],   # 每轮训练损失
    "val_loss":   [float],   # 每轮验证损失
    "val_iou":    [float],   # 每轮验证 IoU
    "val_acc":    [float],   # 每轮验证像素准确率
}
```

### 6.6 visualize.py — 可视化

所有图片统一保存到 `figure/` 目录。

#### `denormalize(tensor)`

将模型使用的归一化图像张量还原为可显示的 uint8 数组：

```
[-1, 1] → × 0.5 + 0.5 → [0, 1] → × 255 → [0, 255] uint8
```

- 输入：`(3, H, W)` Tensor
- 输出：`(H, W, 3)` uint8 numpy 数组

#### `visualize_prediction(model, test_loader, device, num_samples=3)`

从测试集随机抽取 `num_samples` 张图像，并排展示三列：

```
|  Input Image  |  Ground Truth Mask  |  Predicted Segmentation Mask  |
```

保存至 `figure/prediction_result.png`。

#### `plot_training_history(history)`

绘制三个子图并排展示：

| 子图 | 内容 |
|---|---|
| Loss Curve | Train Loss + Val Loss 随 Epoch 变化 |
| IoU Curve | Val IoU 随 Epoch 变化 |
| Pixel Accuracy Curve | Val Pixel Accuracy 随 Epoch 变化 |

保存至 `figure/training_history.png`。

#### `predict_custom_images(model, image_paths, device)`

对自定义图片进行推理并可视化。**自动检测 GT：**

```
图片路径: CamVid/test/xxx.png
    │
    ▼  拼接 "annot" 后缀
标签路径: CamVid/testannot/xxx.png
    │
    ├─ 存在且为单通道灰度图 → 有 GT → 三列布局（Input / GT / Prediction）
    └─ 不存在或为 RGB 图像  → 无 GT → 两列布局（Input / Prediction）
```

保存至 `figure/custom_prediction.png`。

## 7. 数据集说明

### CamVid 数据集

CamVid（Cambridge-driving Labeled Video Database）是道路场景语义分割的标准基准数据集。

### 目录结构

```
CamVid/
├── train/          # 367 张训练图像 (RGB, 360×480)
├── trainannot/     # 367 张训练标签 (灰度索引图, 360×480)
├── val/            # 101 张验证图像
├── valannot/       # 101 张验证标签
├── test/           # 233 张测试图像
├── testannot/      # 233 张测试标签
├── train.txt       # 训练集文件列表
├── val.txt         # 验证集文件列表
└── test.txt        # 测试集文件列表
```

### 标签格式

标签为**单通道灰度索引图**（`PIL mode='L'`），每个像素值代表一个语义类别：

| 索引 | 类别 | 中文 |
|---|---|---|
| 0 | Void | 空/天空 |
| 1 | Building | 建筑 |
| 2 | Column_Pole | 柱子/电线杆 |
| **3** | **Road** | **道路（本项目目标类别）** |
| 4 | Sidewalk | 人行道 |
| 5 | Tree | 树木 |
| 6 | Sign | 标志 |
| 7 | Fence | 围栏 |
| 8 | Car | 汽车 |
| 9 | Pedestrian | 行人 |
| 10 | Cyclist | 骑车人 |
| 11 | Void | 空 |

### 二值化处理

本项目将 12 类多类别分割简化为二值分割：

- **道路（Road, index=3）→ 标签 1（白色）**
- **其他所有类别 → 标签 0（黑色）**

### 数据集下载

- Kaggle: https://www.kaggle.com/datasets/carlolepelaars/camvid
- 原始来源: http://mi.eng.cam.ac.uk/research/projects/VideoRec/CamVid/
