## 2026-05-29 三、根据实际 CamVid 数据集进行的适配修改

### 发现的数据集实际情况

通过读取 `CamVid/` 目录，发现实际数据集与初始代码预期存在多处差异：

| 项目 | 初始代码预期 | 实际数据集 |
|---|---|---|
| 标签目录名 | `{split}_labels/`（如 `train_labels/`） | `{split}annot/`（如 `trainannot/`） |
| 标签格式 | RGB 彩色掩膜（三通道） | **灰度索引图**（单通道 uint8，值 0-11） |
| 道路类别识别 | 通过 RGB 颜色距离匹配 | **类别索引 3 = Road** |
| class_dict.csv | 存在，用于读取颜色定义 | **不存在** |
| 文件名匹配 | 标签带 `_L` 后缀 | **图像与标签同名** |

数据集规模：训练集 367 张，验证集 101 张，测试集 233 张。

### 对 `dataset.py` 的修改

1. **移除 `class_dict.csv` 相关逻辑：** 删除 `get_road_rgb()` 函数和 `pandas` 导入，改用常量 `ROAD_CLASS_INDEX = 3` 硬编码道路类别索引。

2. **修正标签目录名：** 将 `f"{split}_labels"` 改为 `f"{split}annot"`（如 `train` → `trainannot`）。

3. **简化文件名匹配：** 移除 `_L` 后缀等多余的文件名猜测逻辑，改为直接使用同名文件匹配。

4. **重写掩膜转换逻辑：** 将 `_mask_to_binary()` 从 RGB 颜色距离匹配改为灰度索引判等（`mask == ROAD_CLASS_INDEX`）。

5. **修复 Tensor/Numpy 兼容问题：** `ToTensorV2` 会将 mask 也转为 PyTorch Tensor，添加类型检查后先转回 numpy 再进行二值化处理。

### 对 `main.py` 的修改

1. 更新文件头部注释中的数据集目录结构说明，反映实际的 `trainannot/` 命名。
2. 移除不再需要的 `pandas` 安装指令。

### 实现细节

- `visualize.py` 新增 `predict_custom_images()` 函数，负责推理与可视化。
- 自动检测图片是否来自 CamVid 数据集（查找 `testannot/`、`valannot/`、`trainannot/` 标签目录），有 GT 则三列展示（Input / GT / Prediction），无 GT 则两列展示（Input / Prediction）。
- 首次运行时自动创建 `images/` 目录并提示用户放入图片。

### GT 全黑 Bug 修复

**Bug 现象：** 运行 `images/` 目录中的图片时，GT 掩膜列显示为全黑。

**根因分析：** 最初的 GT 查找逻辑使用 `.replace("test", "testannot")` 的字符串替换方式。当图片放在 `images/` 目录时，`"images"` 没有被替换，导致查找路径解析到了**原图自身**（RGB 三通道）。后续 `gt == 3` 在 RGB 数组上几乎找不到匹配像素，因此显示为全黑。

**修复方式：** 改为直接拼接 `{dirname}annot` 后缀（如 `test` + `annot` = `testannot`），并增加单通道维度校验（`len(gt.shape) == 2`），确保只有真正的灰度索引标注才会被作为 GT 展示，彻底避免误匹配到 RGB 原图。

修复后两种场景均验证通过：

| 场景 | 列数 | 结果 |
|---|---|---|
| `CamVid/test/` 路径（有标注） | 3 列 | Input → GT → Prediction，GT 正确显示道路区域 |
| `images/` 路径（无标注） | 2 列 | Input → Prediction，不会误匹配 |

## 2026-06-01 九、Cityscapes GT 全黑 Bug 修复 & 说明文档更新

### GT 全黑 Bug（第二回）

**Bug 现象：** 训练 Cityscapes 后，测试预测可视化中的 Ground Truth 列显示为全黑。CamVid 正常。

**诊断过程：**

逐步检查 Cityscapes 标签数据：

| 检查步骤 | 数据集 | 结果 |
|---|---|---|
| 原始 `_gtFine_labelIds.png` 的像素值 | train/val | 包含 Road=7，占比约 30-40% ✅ |
| 原始 `_gtFine_labelIds.png` 的像素值 | **test** | **仅含值 0、1、3，无 Road=7** ❌ |
| 经过 Resize + Normalize + ToTensorV2 后 | train/val | 值保留正确 ✅ |
| 经过 `_mask_to_binary(mask == 7)` 后 | train/val | 正确生成二值掩膜 ✅ |
| 经过 `_mask_to_binary(mask == 7)` 后 | **test** | **全零（mask 中没有值 7）** ❌ |

**根因分析：** Cityscapes 官方不公开测试集的完整语义标注。`gtFine/test/` 下的 `_gtFine_labelIds.png` 文件仅包含粗略标注（值 0、1、3），不包含道路类别索引 7。因此 `mask == 7` 得到全零数组，GT 显示为全黑。

**修复方式：** 将预测可视化从使用 test 集改为使用 **val 集**。val 集有完整的细粒度标注，Road=7 正常存在。

| 文件 | 改动 |
|---|---|
| `train.py` | `train()` 返回 `val_loader` 而非 `test_loader` |
| `main.py` | `do_train()` 和 `do_test()` 均使用 val 集进行可视化 |

**验证结果：**
- ✅ Val 集 mask 有 Road=1.0，占比约 30%
- ✅ 预测图正常生成（1.5MB，非全黑）

## 2026-06-03 十二、新增 SegNet 模型（U-Net vs SegNet 对比实验架构重构）

### 需求背景

在 CamVid 和 Cityscapes 两个数据集上进行 U-Net 与 SegNet 的对比实验。需要将项目从"单模型 + 双数据集"扩展为"双模型 × 双数据集"的二维架构，并新增完整的对比评估体系。

### U-Net 与 SegNet 的核心区别

| 对比项 | U-Net | SegNet |
|---|---|---|
| 解码器上采样 | 转置卷积（ConvTranspose2d，学习上采样权重） | 最大池化索引（MaxUnpool2d，复用编码器池化位置） |
| 跳跃连接 | 拼接编码器完整特征图（channel concat） | 仅传递池化索引（不传递特征本身） |
| 参数量 | 31,037,633 (31M) | 28,325,377 (28M) |
| 显存占用 | 较高（需存储完整编码器特征图） | 较低（只存池化索引） |

### 对比评估指标体系（11 项）

| # | 指标 | 说明 | 状态 |
|---|---|---|---|
| 1 | **Loss** | BCE + Dice 组合损失 | ✅ 已有 |
| 2 | **IoU** | 交并比 | ✅ 已有 |
| 3 | **Pixel Accuracy** | 像素准确率 | ✅ 已有 |
| 4 | **Precision** | TP/(TP+FP)，预测为道路中真正是道路的比例 | ❌ 新增 |
| 5 | **Recall** | TP/(TP+FN)，真实道路被正确找出的比例 | ❌ 新增 |
| 6 | **F1-Score** | Precision 和 Recall 的调和平均 | ❌ 新增 |
| 7 | **NLL** | 负对数似然，衡量概率预测质量 | ❌ 新增 |
| 8 | **ECE** | 期望校准误差，衡量模型置信度与实际准确率的匹配程度 | ❌ 新增 |
| 9 | **GPU 显存占用** | `torch.cuda.max_memory_allocated()` 峰值显存 | ❌ 新增 |
| 10 | **推理速度 (FPS)** | 每秒处理图片数 | ❌ 新增 |
| 11 | **单 epoch 训练耗时** | 每个 epoch 的训练时间 | ❌ 新增 |

### `model.py` 新增 SegNet 详解

SegNet 编码器与 U-Net 相同（Conv→BN→ReLU→MaxPool），解码器使用 `MaxUnpool2d`：

```python
# 编码器：卷积后 MaxPool 记录索引
e1, idx1 = self.enc1(x)    # 下采样 + 记录池化位置

# 解码器：用索引恢复分辨率，再做卷积
d1 = self.dec1(d2, idx1, output_size=x.size())  # 用 idx1 反向恢复
```

关键组件：
- `SegNetDown`：Conv×2 → BN → ReLU → MaxPool(return_indices=True)
- `SegNetUp`：MaxUnpool(indices) → Conv×2 → BN → ReLU
